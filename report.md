# Part B 实验报告

## 1. 实验设置

本实验使用提供的 `train.jsonl` 和 `valid.jsonl`，数据格式是 chat 格式。训练集有 219 条样本，验证集有 17 条样本。基座模型使用从 ModelScope 下载的 `google/gemma-3-270m-it`，微调方式是基于 PEFT 的 LoRA。

这个任务的主要限制是模型非常小。Gemma-3-270M 只有 2.7 亿参数，和生成参考 SVG 的强模型差距很大，所以我的目标不是生成很漂亮的徽标，而是先让模型尽量输出“有效、闭合、结构简单”的 SVG。语义匹配只占较小权重。

## 2. Reward 设计

`reward.py` 中的 reward 是一个 `[0, 1]` 之间的程序化代理分数，用来衡量生成 SVG 的结构质量。

| 评分项 | 权重 | 设计理由 |
|---|---:|---|
| SVG 结构 | 0.30 | 检查是否有 `<svg>` 和 `</svg>`，XML 是否能解析，根节点是否是 SVG，以及 viewBox 是否合理。这是小模型最需要先学会的能力。 |
| 图形元素数量 | 0.17 | 奖励非空 SVG，并要求有适量的基础图形元素。空 SVG 或大量重复堆砌元素会被扣分。 |
| 配色数量 | 0.12 | 徽标通常需要一个小而稳定的配色方案。单色可以接受但分数较低，颜色过多通常表示输出混乱。 |
| 坐标合理性 | 0.12 | 惩罚明显越界的坐标和不合理尺寸，同时对正常 path 坐标保持一定容忍。 |
| 标签和属性安全性 | 0.09 | 禁止脚本、foreignObject、外部链接、事件处理器等不适合本任务的内容。 |
| 填充 path 闭合性 | 0.06 | 有填充色的 path 通常应该以 `Z` 闭合；只有描边的装饰线条可以不闭合。 |
| 退化输出检查 | 0.09 | 惩罚 markdown 代码块、重复 SVG、过短或过长输出、placeholder、危险标签，以及重复图形片段。 |
| 提示词关键词覆盖 | 0.05 | 对颜色词和常见主题词的匹配给少量加分。这个权重较低，因为很多视觉元素在 SVG 中只表现为 path 坐标。 |

最终版本的 reward 对 Sonnet 参考答案给分较高：验证集参考 SVG 的平均分约为 `0.920`。这说明该 reward 至少在结构层面和数据集目标比较一致。

## 3. 训练配置

为了适配本机 6GB GTX 1660 SUPER，我使用了比较保守的 LoRA 配置：

- LoRA rank：8
- LoRA alpha：16
- LoRA dropout：0.05
- 目标模块：attention projection 和 MLP projection
- 学习率：`1e-4`
- 训练轮数：2
- batch size：1
- 梯度累积：8
- 最大序列长度：1536
- 精度：fp32。原因是这张 GTX 1660 SUPER 上用 fp16 会出现 NaN loss
- 验证集贪心解码：`max_new_tokens=700`，`repetition_penalty=1.1`
- prompt masking：system/user prompt 对应的 label 设为 `-100`，loss 只计算 assistant 的 SVG 部分

完整超参数见 `train_config.yaml`。

## 4. 自评结果

最终自评结果由下面命令生成，并写入 `results.json`：

```powershell
.\.venv\Scripts\python.exe student_kit\eval_self.py --config train_config.yaml --output results.json
```

验证集上的主要结果如下：

| 模型 | 平均 reward | 有效 XML 比例 | 闭合 SVG 比例 | 说明 |
|---|---:|---:|---:|---|
| 参考 SVG | 0.920 | 1.000 | 1.000 | Sonnet 生成的目标答案，只作为结构上限参考。 |
| 基座 Gemma-3-270M | 0.430 | 0.353 | 0.471 | 方差很大：少数样本能生成有效 SVG，但很多样本是 malformed 或很短的输出。 |
| LoRA adapter | 0.416 | 0.235 | 0.588 | 更容易输出闭合的 SVG 风格文本，但仍经常重复形状或破坏 XML 属性。 |

从平均分看，LoRA 没有超过基座模型，mean delta 是 `-0.014`。不过也有一些局部改善：中位数从 `0.250` 提高到 `0.330`，最低分从 `0.102` 提高到 `0.197`，闭合 SVG 比例从 `0.471` 提高到 `0.588`。

因此，这次微调不是一个明显成功的结果。它让最差的一部分输出不再那么空或短，但没有让整体 reward 超过基座。主要原因是基座模型在固定解码设置下偶尔也能生成结构分很高的 SVG，而 LoRA adapter 虽然学到了一些 SVG 开头和闭合模式，但自由生成时仍容易重复。

## 5. 结果分析

训练 loss 和验证 loss 都是正常下降的，最终验证 loss 约为 `0.909`。这说明模型在 teacher forcing 条件下确实学到了一些 SVG token 分布。但是自由生成时，模型经常先生成一个看起来合理的 SVG header，然后开始重复简单的 `<rect>` 或 `<circle>`，或者生成格式错误的属性，最后即使有 `</svg>` 也不一定是有效 XML。

这也解释了为什么 LoRA 的闭合 SVG 比例提高了，但有效 XML 比例反而下降：模型学会了“继续写 SVG 风格文本”和“有时闭合”，但还没有稳定学会全局语法一致性。

这个 reward 也存在被利用的问题。例如，模型输出一个有效 SVG，里面只有几个通用圆形，也可能拿到不错的结构分，但它不一定真正符合提示词。这是典型的 Goodhart 现象：代理指标上去了，不代表真实视觉质量一定上去。私有评测中包含冻结指标和视觉评审，应该能更好地区分这种情况。

本次提交中的 reward 有意保持简单和鲁棒，没有把关键词匹配权重设得很高。原因是 SVG 的视觉语义很难只靠文本关键词判断，尤其很多元素都藏在 path 坐标里。

## 6. 代表样例

从 `results.json` 中选三个验证样例：

1. Index 1：adapter score 为 `0.865`，XML 有效且 SVG 闭合。这个样例有比较完整的 SVG 外壳，但仍然依赖重复的简单背景形状。
2. Index 3：adapter score 为 `0.197`，输出 malformed 且没有正确闭合。这是 LoRA 后常见失败模式：能延续 SVG 风格语法，但属性格式和整体结构仍不稳定。
3. Index 0：base score 为 `0.919`，adapter score 为 `0.848`。这个样例说明为什么平均 delta 为负：基座模型在某些样本上也可能击中当前结构代理指标。

如果继续改进，我会优先尝试两件事：第一，截断或筛选训练目标，只保留更短、更紧凑的 SVG；第二，先做一个只学习有效闭合 SVG 的 curriculum，再逐步加入更复杂的图形和语义要求。当前训练集中的参考 SVG 对 270M 小模型来说偏长，模型更容易先学到局部 SVG 语法，而不是全局闭合和提示词保真度。
